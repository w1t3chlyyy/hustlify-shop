# Hustlify — магазин + оплата + Telegram-бот + админка

## Структура проекта
```
hustlify-shop/
  server.js              ← бэкенд (Express) — весь API в одном файле
  telegram.js             ← бот-уведомитель о заказах (файл с кодом бота)
  get-chat-id.js            ← вспомогательный скрипт: находит ваш chat_id
  package.json
  .env.example              ← скопируйте в .env и заполните своими ключами
  data/
    products.json            ← товары и кейсы (редактируются через админку)
    news.json                 ← новости на главной (редактируются через админку)
    orders.json                ← заказы
    promocodes.json             ← промокоды из рулетки
  public/
    index.html                   ← сам магазин (то, что видят покупатели)
    admin.html                    ← админ-панель (то, что видите только вы)
    order-success.html             ← страница "спасибо за заказ"
    images/                          ← сюда кладите фото товаров
```

## Запуск локально
```bash
cd hustlify-shop
npm install
cp .env.example .env
```
Откройте `.env` и заполните:
1. **ADMIN_LOGIN / ADMIN_PASSWORD_HASH** — ваш логин и хеш пароля в админку.
   Сгенерировать хеш:
   ```bash
   node -e "console.log(require('bcryptjs').hashSync('ваш_пароль', 10))"
   ```
   Скопируйте результат в `ADMIN_PASSWORD_HASH`.
2. **JWT_SECRET** — любая случайная длинная строка (например, `openssl rand -hex 32`).
3. **CRYPTOBOT_TOKEN** — получите в Telegram-боте `@CryptoBot` → «Crypto Pay» → «Create App».
4. **YOOKASSA_SHOP_ID / YOOKASSA_SECRET_KEY** — из личного кабинета yookassa.ru → Настройки → API.
5. **TELEGRAM_BOT_TOKEN / TELEGRAM_ADMIN_CHAT_IDS** — см. раздел про бота ниже.
6. **PUBLIC_URL** — адрес сайта после деплоя (локально подойдёт `http://localhost:3000`, но вебхуки платёжек и Telegram работают только с публичным HTTPS-доменом).

Запуск:
```bash
npm start
```

## 🔑 Где найти страницу с админкой
Админка — это отдельная страница `admin.html`, которая лежит рядом с магазином на том же сервере:

- **Пока тестируете локально:** `http://localhost:3000/admin.html`
- **После деплоя на Render (до подключения своего домена):** `https://ваш-сервис.onrender.com/admin.html`
- **С собственным доменом:** `https://ваш-домен.ru/admin.html`

Она никак не связана и не доступна из самого магазина (в меню/футере ссылки на неё нет специально) — просто откройте этот адрес напрямую в браузере и сохраните в закладки. Вход — по логину и паролю из `ADMIN_LOGIN` / `ADMIN_PASSWORD_HASH` в вашем `.env` (см. пункт 1 выше — это не пароль "по умолчанию", вы сами его придумываете при настройке).

Если увидите на странице форму логина — значит всё работает правильно, просто введите свои данные.

## Что можно делать в админке
Три вкладки:
- **«Товары и кейсы»** — добавление/редактирование/удаление любых позиций каталога. Переключатель «Все / Каталог / Кейсы» наверху фильтрует список — то, что на сайте показано в блоке «Кейсы проекта Hustlify», это те же товары с разделом «Кейс», редактируются в этой же вкладке.
- **«Новости»** — карточки в блоке «Новостной поток» на главной. Добавляйте, редактируйте, удаляйте — сайт сразу подтянет изменения.
- **«Заказы»** — список всех заказов: состав, сумма, контакт покупателя, применённый промокод, статус оплаты. Можно вручную отметить заказ оплаченным.

Всё, что вы меняете в админке, сразу видно на сайте у всех посетителей — данные хранятся на сервере, а не в браузере.

## Telegram-бот с уведомлениями о заказах
Код бота лежит в `telegram.js` — это не отдельная программа, а модуль, который `server.js` подключает и вызывает при создании и оплате заказа. Отдельно "запускать" бота не нужно, он живёт внутри основного сервера.

Настройка (2 минуты):
1. В Telegram напишите **@BotFather** → `/newbot` → следуйте подсказкам → получите токен вида `123456:AAExxxxxxx`.
2. Впишите его в `.env` как `TELEGRAM_BOT_TOKEN`.
3. Напишите вашему новому боту любое сообщение (например "привет") — иначе он не сможет писать вам первым.
4. Узнайте свой chat_id одной командой:
   ```bash
   npm run chat-id -- ВАШ_ТОКЕН_БОТА
   ```
   Скрипт покажет `chat_id` — впишите его в `.env` как `TELEGRAM_ADMIN_CHAT_IDS` (через запятую, если получателей несколько, например вы и менеджер).
5. Перезапустите сервер (`npm start`).

С этого момента бот присылает сообщение:
- 🆕 когда создан новый заказ (ещё не оплачен) — состав, сумма, промокод (если был), контакт покупателя;
- ✅ когда заказ оплачен — автоматически по вебхуку CryptoBot/ЮKassa, либо когда вы вручную отмечаете оплату в админке.

## Контакты покупателя и промокоды — как это работает
- При оформлении заказа покупатель видит окно с полем **«Как с вами связаться»** и полем **«Промокод»** — это попадает и в заказ, и в Telegram-уведомление.
- Промокоды не выдумываются в браузере: рулетка (`POST /api/promo/generate`) генерирует код **на сервере** и сохраняет его в `data/promocodes.json` со сроком действия 24 часа.
- **Код не содержит внутри себя размер скидки.** Это случайная непрозрачная строка вроде `R7QK9M2XPLTN` (12 символов из 32-буквенного алфавита) — по её виду нельзя понять или подделать процент скидки. Сама скидка хранится только в базе на сервере и подставляется по коду. Подобрать код перебором нереально, плюс лимит — не чаще одного раза в 5 секунд с одного IP.
- При оформлении заказа сервер сам проверяет код: не использован ли, не истёк ли — и применяет скидку к сумме. Код одноразовый, сгорает сразу после применения.
- В админке во вкладке «Заказы» видно, какой промокод и с какой скидкой был применён, плюс контакт покупателя.

## Деплой на Render.com

⚠️ **Важная особенность Render, из-за которой обычный «бесплатный» тариф не подойдёт**: у бесплатных веб-сервисов Render файловая система эфемерная — все файлы стираются при каждом перезапуске/передеплое, а подключить постоянный диск можно только на платном тарифе. Наш магазин хранит товары/заказы/промокоды в файлах на диске, поэтому нужен план **Starter** ($7/мес) + **Persistent Disk** ($0.25/ГБ/мес, для наших json хватит 1 ГБ = 25 центов) — итого около **$7.25/мес**. Я уже подготовил код под это (переменная `DATA_DIR`), просто следуйте шагам ниже.

### Шаг 1. Выложите проект на GitHub
Render разворачивает только из Git-репозитория.
```bash
cd hustlify-shop
git init
git add .
git commit -m "Hustlify shop"
```
Создайте пустой репозиторий на [github.com/new](https://github.com/new), затем:
```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/hustlify-shop.git
git branch -M main
git push -u origin main
```
`.env` в репозиторий не попадёт — он уже в `.gitignore`. Это правильно: ключи вводятся прямо в Render, а не хранятся в коде.

### Шаг 2. Создайте Web Service
1. Зарегистрируйтесь на [render.com](https://render.com) (можно через GitHub).
2. **New → Web Service** → подключите ваш GitHub-репозиторий `hustlify-shop`.
3. Настройки:
   - **Environment**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: **Starter** ($7/мес) — не Free (см. предупреждение выше)

### Шаг 3. Подключите постоянный диск
Ещё на экране создания (или потом во вкладке **Disks**):
- **Add Disk** → Name: `data`, **Mount Path**: `/var/data`, **Size**: 1 GB

### Шаг 4. Переменные окружения
В разделе **Environment** добавьте те же переменные, что и в `.env.example`, плюс новую:
```
DATA_DIR=/var/data
PORT=10000
PUBLIC_URL=https://ВАШ-СЕРВИС.onrender.com   (обновите после первого деплоя, см. шаг 5)
ADMIN_LOGIN=...
ADMIN_PASSWORD_HASH=...
JWT_SECRET=...
CRYPTOBOT_TOKEN=...
CRYPTOBOT_ASSET=USDT
YOOKASSA_SHOP_ID=...
YOOKASSA_SECRET_KEY=...
TELEGRAM_BOT_TOKEN=...
TELEGRAM_ADMIN_CHAT_IDS=...
```
(Render сам прокидывает свой `PORT` — можно не указывать вручную, но `10000` его значение по умолчанию.) Наш сервер уже слушает `process.env.PORT` и все интерфейсы — код под Render менять не нужно.

### Шаг 5. Деплой и финальная настройка
1. **Create Web Service** — Render соберёт и запустит проект (2–5 минут). В логах увидите `Инициализирован products.json на постоянном диске...` — это разовое копирование стартовых товаров/новостей на ваш новый диск.
2. Скопируйте выданный адрес вида `https://hustlify-shop.onrender.com`.
3. Обновите переменную `PUBLIC_URL` этим адресом (или вашим кастомным доменом, см. ниже) → Render передеплоит сервис автоматически.
4. В кабинетах CryptoBot и ЮKassa укажите webhook-адреса:
   - `https://ВАШ-АДРЕС/api/webhooks/cryptobot`
   - `https://ВАШ-АДРЕС/api/webhooks/yookassa`
5. Админка: `https://ВАШ-АДРЕС/admin.html`

### Свой домен вместо onrender.com
В настройках сервиса → **Custom Domains** → добавьте домен → Render покажет DNS-запись (CNAME/A), которую нужно прописать у вашего регистратора домена. TLS-сертификат Render выпускает и обновляет сам, бесплатно. После подключения домена не забудьте обновить `PUBLIC_URL` и webhook-адреса на новый домен.

### Дальше — само собой
Каждый `git push` в `main` автоматически пересобирает и передеплоивает сайт. Логи и статус — во вкладке **Logs** в панели Render.

### Что важно понимать про Starter-тариф
На платном Starter-инстансе сервис не «засыпает» и не имеет задержек при вебхуках от платёжек — это и есть причина выбрать его вместо Free для магазина, где проходят реальные деньги. Free-тариф можно использовать только для черновой проверки, что деплой в принципе работает — но без диска Free-сервис забудет все товары/заказы при следующем перезапуске (они происходят регулярно, а не только руками), так что для боевого запуска он не годится.

### Альтернатива: обычный VPS
Если предпочитаете VPS (Timeweb, Selectel, Hetzner) вместо Render:
1. Разместите проект на сервере, укажите `PUBLIC_URL=https://ваш-домен.ru` в `.env`. `DATA_DIR` можно не задавать — на VPS диск обычный, не эфемерный, подойдёт папка `./data` по умолчанию.
2. Настройте HTTPS (Let's Encrypt / Certbot, если Nginx стоит перед Node).
3. Вебхуки — те же пути: `/api/webhooks/cryptobot` и `/api/webhooks/yookassa`.
4. Запустите процесс через `pm2`, чтобы сервер работал постоянно и перезапускался при сбое:
   ```bash
   npm install -g pm2
   pm2 start server.js --name hustlify
   pm2 save
   ```

## Добавление товаров, кейсов, новостей и фото
Всё через админку — see «Что можно делать в админке» выше.
Для фото: положите файл в `public/images/`, в поле «Ссылка на фото» укажите `/images/имя-файла.png`.

## Важно про CryptoBot
Инвойс создаётся в выбранной валюте (`CRYPTOBOT_ASSET`, по умолчанию USDT) с суммой, равной сумме заказа в рублях "как есть". Перед боевым запуском добавьте конвертацию рубль → USDT/TON по актуальному курсу (например, через любой публичный курс-API), иначе клиенты будут платить неверную сумму в валюте.

## Важно про безопасность
- Никогда не публикуйте `.env` (он уже в `.gitignore`, если используете git — не удаляйте эту строку).
- Пароль админки хранится только как bcrypt-хеш — сам пароль нигде не сохраняется в открытом виде.
- Файловая база (`data/*.json` или `DATA_DIR` на постоянном диске) подходит для старта; при заметном росте заказов стоит перейти на настоящую БД (PostgreSQL/SQLite) — структура кода это позволяет сделать без переписывания фронтенда.
- На Render (или другой платформе с эфемерным диском) обязательно настройте persistent disk и `DATA_DIR` — иначе при каждом перезапуске сервиса пропадут все товары, заказы и промокоды. Подробности — в разделе «Деплой на Render.com».
